Keep your notes private.

A small CGI application that shows how server-side authentication keeps everyone's data separate — built with HTML, C, Apache, and plain CSV files. No database, no frameworks, just a login and a filter.

How it works

Every note is stored with the ID of the person who wrote it. The server only ever shows you the rows that belong to you.

STEP 1

Enter credentials

The visitor types a username and password into the login form.

STEP 2

Submit to the server

The browser sends the form to login.cgi, a C program running on Apache.

STEP 3

Check the user

login.cgi opens users.csv and asks: does this user exist?

STEP 4

Establish identity

On success, the program remembers who's asking: current_user = 1.

STEP 5

Read every note

The program opens notes.csv and reads every row in the file, one by one.

STEP 6

Filter and display

Only rows where user_id == current_user are sent back to the browser.
